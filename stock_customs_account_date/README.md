# stock_customs_account_date — Stock Customs Account Date

## Overview

This module extends Odoo's `stock.picking` and `stock.move` models to allow users to define a custom accounting date (referred to as "customs date") on stock transfers. When this date is set, it forces the accounting entries generated during stock valuation to use that specific date instead of the transfer's actual done date. Additionally, for moves linked to purchase orders in a foreign currency, the price unit is recalculated using the exchange rate corresponding to the customs accounting date.

## Dependencies

### Odoo / OCA Modules

- `stock_account` — Required to access stock valuation account move generation (`_create_account_move_line`, `_get_price_unit`) and the stock picking form view being extended.
- `purchase_stock` — Required to access purchase order line data (`purchase_line_id`) on stock moves for foreign currency price unit recalculation.

### Python Libraries

None.

## Installation

1. Place the module in your addons path.
2. Update the module list: **Settings → Activate Developer Mode → Update Apps List**
3. Search for `stock_customs_account_date` and click **Install**.
4. No post-install configuration steps are required. The `accounting_date` and `exchange_rate` fields will be immediately available on all stock transfer forms.

## Models Reference

### `stock.picking` — Transfer

| Field Name | Type | Label (string) | Description |
|---|---|---|---|
| `accounting_date` | `Date` | Accounting Date | Custom date used to force the accounting period for journal entries generated by this transfer. `copy=False` ensures it is not duplicated when the picking is copied. |
| `exchange_rate` | `Float` (digits 12,4) | Exchange Rate | Computed field that returns the USD→MXN conversion rate for the `accounting_date` (or today if no accounting date is set). Depends on `accounting_date`. |

**Key methods:**

- `_compute_exchange_rate`: Computes the USD to MXN exchange rate using `res.currency._get_conversion_rate` for the company, using `accounting_date` if set or today's date as fallback.

---

### `stock.move` — Stock Move

| Field Name | Type | Label (string) | Description |
|---|---|---|---|
| *(no new fields added)* | — | — | This model is extended via method overrides only. |

**Key methods:**

- `_create_account_move_line`: Overrides the standard method. If the related picking has an `accounting_date` set, it injects `force_period_date` into the context with that date before calling `super()`, forcing the resulting journal entry to be posted on that date.
- `_get_price_unit`: Overrides the standard method. If the related picking has an `accounting_date` and the move is linked to a purchase order line (`purchase_line_id`), it recalculates the price unit by: (1) extracting the PO line unit price, (2) stripping taxes to get the tax-excluded price, (3) converting UoM if different from the product's base UoM, (4) converting from the PO currency to the company currency using the exchange rate at `accounting_date`. It then writes the recalculated `price_unit` and `date` onto the move before returning. Falls back to `super()` if no accounting date or no purchase line is present.

## Security Groups

No new security groups or access control rules are defined in this module. Access follows the standard permissions of the `stock_account` and `purchase_stock` modules.

## Menus & Actions

No new menu items or actions are defined in this module. The functionality is exposed by adding fields directly to the existing **Inventory > Operations > Transfers** form view (the native `stock.view_picking_form`).

The two new fields appear on the transfer form immediately after the **Effective Date** (`date_done`) field:

- **Accounting Date** — editable when the transfer is not in `done` or `cancel` state.
- **Exchange Rate** — read-only computed field displayed alongside the accounting date.

## Descripción General

Este módulo fue desarrollado para empresas que importan mercancía y necesitan controlar con precisión la fecha contable que se utiliza al generar los asientos de valoración de inventario. En México, y en general en operaciones de comercio exterior, la fecha de la póliza contable no siempre coincide con la fecha en que físicamente se recibe la mercancía: el pedimento aduanal, la factura del proveedor y la entrada al almacén pueden ocurrir en días distintos.

Sin este módulo, Odoo genera los asientos contables usando la fecha real de la transferencia (fecha de validación). Esto puede ocasionar que el movimiento quede registrado en un período fiscal diferente al que corresponde según la documentación aduanal, causando discrepancias en los reportes contables y problemas al cerrar períodos.

Con este módulo, el operador o el área de contabilidad puede indicar directamente en la transferencia de almacén una **Fecha Contable** específica. Al validar la entrada, todos los asientos de inventario se generarán con esa fecha, independientemente de cuándo ocurrió físicamente el movimiento. Además, si la compra fue en moneda extranjera (por ejemplo, dólares americanos), el sistema recalcula automáticamente el tipo de cambio correspondiente a esa fecha contable, garantizando que el valor en pesos mexicanos sea el correcto según la normativa fiscal.

## Objetivo de Negocio

- Permitir que las transferencias de inventario, especialmente las recepciones de importación, queden registradas contablemente en la fecha del pedimento aduanal y no en la fecha de recepción física.
- Garantizar que el tipo de cambio utilizado en la valuación del inventario corresponda a la fecha del documento aduanal, cumpliendo con las disposiciones fiscales mexicanas.
- Evitar discrepancias entre los registros contables y la documentación de comercio exterior.
- Facilitar el cierre de períodos contables sin necesidad de ajustes manuales por diferencias de fecha.
- Brindar visibilidad del tipo de cambio USD/MXN vigente para la fecha contable directamente en la pantalla de la transferencia.

## Flujos de Negocio Principales

### Flujo 1: Recepción de mercancía importada con fecha aduanal

**Descripción:** Una empresa recibe mercancía proveniente de un proveedor extranjero. La mercancía llega físicamente al almacén en una fecha, pero el pedimento aduanal tiene una fecha diferente (anterior o posterior). El área de contabilidad necesita que el asiento de valuación de inventario quede con la fecha del pedimento.

**Pasos:**
1. El operador de almacén recibe la mercancía en Odoo y abre la transferencia correspondiente en **Inventario → Operaciones → Transferencias**.
2. Antes de validar la transferencia, el usuario localiza el campo **Fecha Contable** en el formulario (visible después del campo *Fecha Efectiva*).
3. El usuario ingresa la fecha del pedimento aduanal en el campo **Fecha Contable**.
4. El sistema muestra automáticamente el campo **Tipo de Cambio**, reflejando el tipo de cambio USD/MXN para esa fecha.
5. El usuario valida la transferencia normalmente.
6. Odoo genera los asientos contables de valuación de inventario usando la **Fecha Contable** indicada, no la fecha real de validación.
7. Si la orden de compra estaba en dólares, el precio unitario registrado en los asientos utiliza el tipo de cambio de la fecha contable.

**Reglas de negocio importantes:**
- El campo **Fecha Contable** solo es editable mientras la transferencia esté en estado borrador o en proceso. Una vez que la transferencia está en estado *Hecho* o *Cancelado*, el campo queda en modo solo lectura.
- El campo **Fecha Contable** no se copia si se duplica la transferencia (`copy=False`).
- Si no se establece una **Fecha Contable**, el sistema se comporta exactamente igual que el estándar de Odoo (sin cambios en la fecha ni en el tipo de cambio).
- El tipo de cambio mostrado corresponde específicamente a la conversión de USD a MXN.

### Flujo 2: Consulta del tipo de cambio para una fecha de pedimento

**Descripción:** El área de contabilidad o de comercio exterior necesita verificar qué tipo de cambio USD/MXN será aplicado para una fecha de pedimento determinada antes de validar la transferencia.

**Pasos:**
1. El usuario abre la transferencia en **Inventario → Operaciones → Transferencias**.
2. Introduce una fecha en el campo **Fecha Contable**.
3. El campo **Tipo de Cambio** se actualiza automáticamente mostrando el tipo de cambio USD/MXN correspondiente a esa fecha, tomando los datos de los tipos de cambio registrados en Odoo.
4. El usuario puede verificar que el tipo de cambio es el correcto antes de proceder a validar.

**Reglas de negocio importantes:**
- Si no hay **Fecha Contable** definida, el tipo de cambio mostrado corresponde a la fecha de hoy.
- El tipo de cambio es un campo calculado (solo lectura); no puede editarse directamente.

## Guía de Configuración

1. **Instalar el módulo** siguiendo los pasos de instalación indicados en la sección técnica.
2. **Configurar los tipos de cambio** en Odoo: dirigirse a **Contabilidad → Configuración → Divisas** y asegurarse de que la divisa USD (Dólar Americano) y MXN (Peso Mexicano) estén activas y con los tipos de cambio históricos cargados. El módulo consulta directamente esta tabla para calcular el campo **Tipo de Cambio**.
3. **Verificar que las órdenes de compra** en moneda extranjera estén correctamente configuradas con la divisa correspondiente, ya que el recálculo del precio unitario depende de la moneda de la orden de compra.
4. **Capacitar al equipo** de almacén y/o contabilidad para que ingresen la **Fecha Contable** antes de validar las transferencias de importación.
5. **Advertencia:** Si los tipos de cambio históricos no están cargados en Odoo para la fecha indicada en el pedimento, el sistema podría utilizar el tipo de cambio más cercano disponible. Se recomienda mantener actualizados los tipos de cambio históricos.

| Campo en pantalla | Para qué sirve | Valores posibles / Notas |

## Automatizaciones y Reglas

- **Recálculo del tipo de cambio en movimientos de compra:** Se ejecuta automáticamente al momento de generar los asientos de valuación de inventario (`_get_price_unit`), cuando la transferencia tiene una **Fecha Contable** definida y el movimiento de stock está vinculado a una línea de orden de compra. Resultado: el precio unitario del movimiento se recalcula usando el tipo de cambio de la fecha contable y se actualiza en el registro del movimiento antes de generar el asiento.
- **Forzado de fecha en asientos contables:** Se ejecuta automáticamente durante la generación de líneas de asiento contable (`_create_account_move_line`). Si la transferencia tiene **Fecha Contable**, el contexto de Odoo se modifica con `force_period_date` para que el asiento quede en el período correcto. Resultado: todos los asientos de valoración del inventario de esa transferencia reflejan la fecha contable indicada.

## Preguntas Frecuentes (FAQ)

**P: ¿Qué pasa si no lleno el campo "Fecha Contable" antes de validar la transferencia?**
R: Si no se establece una Fecha Contable, el módulo no altera ningún comportamiento. Odoo generará los asientos contables y calculará los precios unitarios exactamente igual que en la versión estándar, usando la fecha real de validación de la transferencia.

**P: ¿Puedo modificar la Fecha Contable después de validar la transferencia?**
R: No. Una vez que la transferencia está en estado *Hecho* o *Cancelado*, el campo Fecha Contable queda en modo solo lectura y no puede modificarse desde la interfaz. Esto es una medida de control para preservar la integridad de los registros contables ya generados.

**P: ¿El Tipo de Cambio que se muestra en pantalla es el que realmente se usará en los asientos?**
R: Sí, siempre y cuando los tipos de cambio históricos estén correctamente cargados en Odoo para esa fecha. El mismo método de conversión (`res.currency._get_conversion_rate`) es el que utiliza el sistema al momento de generar los asientos contables.

**P: ¿Este módulo funciona para compras en cualquier moneda extranjera?**
R: El campo de visualización **Tipo de Cambio** está configurado específicamente para mostrar la conversión de USD a MXN. Sin embargo, el recálculo del precio unitario en los asientos funciona con cualquier moneda extranjera definida en la orden de compra, convirtiendo a la moneda de la compañía usando la fecha contable indicada.

**P: ¿Si duplico una transferencia, se copia también la Fecha Contable?**
R: No. El campo Fecha Contable tiene la propiedad `copy=False`, lo que significa que al duplicar una transferencia, ese campo quedará vacío en la nueva transferencia. Deberás ingresarlo manualmente.

**P: ¿Este módulo afecta las transferencias internas o solo las recepciones de compra?**
R: El campo **Fecha Contable** aparece en todos los tipos de transferencias (recepciones, entregas, transferencias internas). Sin embargo, el recálculo del precio unitario con tipo de cambio solo aplica cuando el movimiento está vinculado a una línea de orden de compra (`purchase_line_id`). Para transferencias sin orden de compra, solo se fuerza la fecha en los asientos si se define la Fecha Contable.

**P: ¿Qué sucede si ingreso una Fecha Contable que corresponde a un período contable ya cerrado?**
R: Odoo validará según las restricciones de bloqueo de períodos configuradas en contabilidad. Si el período está bloqueado, la validación de la transferencia fallará con un mensaje de error de Odoo estándar. Se recomienda coordinar con el área de contabilidad antes de usar fechas de períodos anteriores.

**Lo que este módulo NO hace:**

- No permite editar la Fecha Contable una vez que la transferencia ha sido validada o cancelada.
- No aplica el tipo de cambio personalizado a documentos fuera del flujo de transferencias de almacén (facturas, pagos, etc.).
