## Overview

This module provides a transient wizard (`stock.picking.change.date.wizard`) that allows authorized users to retroactively change the date of validated stock pickings and all their related accounting entries. It extends the native `stock.picking`, `stock.move`, `stock.move.line`, `stock.valuation.layer`, and `account.move` models by updating their date fields in a single coordinated operation. A custom security group gates access to this functionality, restricting it to explicitly authorized users.

## Dependencies

### Odoo / OCA Modules

- `stock_account` — Required to access stock valuation layers (`stock.valuation.layer`) and the accounting entries (`account.move`) generated by inventory movements, which are the primary targets of the date-change operation.

### Python Libraries

None.

## Installation

1. Place the module in your addons path.
2. Update the module list: **Settings → Activate Developer Mode → Update Apps List**
3. Search for `stock_picking_change_date` and click **Install**.
4. After installation, add the desired users to the security group **Allow to change date in picking** via **Settings → Users & Companies → Users** (by default only `Administrator` and `OdooBot` are included).

## Models Reference

### `stock.picking.change.date.wizard` — Wizard to allow to change date to picking and related account move.

| Field Name | Type | Label (string) | Description |
|---|---|---|---|
| `date` | `Datetime` | *(not explicitly set — defaults to field name)* `date` | The target date/time to apply to the selected pickings and all related records. Required. Cannot be a future date. |

**Key methods:**

- `change_date`: Validates that the selected date is not in the future and that all selected pickings are in state `done`; then writes the new date to the pickings (`accounting_date`, `date_done`), their stock moves (`date`), stock move lines (`date`), updates `stock_valuation_layer.create_date` via raw SQL (bypassing ORM restriction), resets and reposts the related `account.move` records with the new date.

## Security Groups

| Group ID | Group Name | Applied To |
|---|---|---|
| `stock_picking_change_date.allow_change_date_picking` | Allow to change date in picking | Grants access to the wizard; by default includes `base.user_admin` and `base.user_root` |

### Access Rights (`ir.model.access.csv`)

| Model | Group | Read | Write | Create | Delete |
|---|---|---|---|---|---|
| `stock.picking.change.date.wizard` | `base.group_user` (all internal users) | ✓ | ✓ | ✓ | ✓ |

> **Note:** All internal users can instantiate the wizard, but the intended usage (invoking it from a picking action) should be further restricted in practice by menu/button visibility tied to the `allow_change_date_picking` group.

## Menus & Actions

This module does not define top-level menu items. The wizard is accessed via a contextual action launched from the **Inventory** application on validated stock pickings. The wizard view is defined in `wizards/stock_picking_change_date_wizard_view.xml` and is triggered as a server action or button action from selected picking records.

## Descripción General

Este módulo resuelve una necesidad operativa y contable recurrente en empresas que utilizan Odoo para gestionar su inventario: la posibilidad de corregir la fecha de una transferencia de almacén ya validada, junto con todos los asientos contables que se generaron a partir de ella. En la operación diaria es común que una transferencia se valide con la fecha incorrecta —por ejemplo, por un retraso en el sistema, un cierre de período contable o un error del operador— y que esto provoque inconsistencias en los reportes financieros y de inventario.

Sin este módulo, cambiar la fecha de una transferencia validada en Odoo requiere cancelar y rehacer la operación completa, con el riesgo de afectar otras transacciones relacionadas. Con este módulo, un usuario autorizado puede seleccionar una o varias transferencias ya completadas y cambiar su fecha mediante un asistente sencillo, propagando el cambio automáticamente a todos los registros relacionados: movimientos de stock, líneas de movimiento, capas de valoración de inventario y asientos contables.

El módulo está diseñado para el área de **Operaciones / Almacén** y el área de **Contabilidad**, y su acceso está restringido a usuarios con permisos especiales para evitar modificaciones no autorizadas en registros históricos.

## Objetivo de Negocio

- Permitir la corrección de fechas en transferencias de almacén ya validadas sin necesidad de cancelar y rehacer la operación.
- Mantener la consistencia entre los registros de inventario y los asientos contables asociados al modificar la fecha en todos los modelos relacionados de forma simultánea.
- Garantizar que las capas de valoración de inventario reflejen la fecha correcta para el cálculo de costos y reportes financieros.
- Restringir esta operación sensible únicamente a usuarios autorizados mediante un grupo de seguridad específico.
- Proteger la integridad contable impidiendo el uso de fechas futuras en la corrección.

## Flujos de Negocio Principales

### Flujo 1: Cambio de Fecha en Transferencias Validadas

**Descripción:** Se activa cuando un usuario autorizado detecta que una o varias transferencias de almacén en estado "Hecho" tienen una fecha incorrecta y necesita corregirla para que los reportes de inventario y contabilidad reflejen la realidad operativa.

**Pasos:**

1. El usuario navega al módulo **Inventario** y localiza la(s) transferencia(s) que requieren corrección de fecha. Todas deben estar en estado **Hecho**.
2. El usuario selecciona una o varias transferencias usando las casillas de selección en la vista de lista.
3. El usuario ejecuta la acción **Cambiar Fecha** (disponible para usuarios con el grupo *Allow to change date in picking*), lo que abre el asistente.
4. En el asistente, el usuario ingresa la **fecha y hora** correcta a aplicar.
5. El usuario confirma haciendo clic en el botón de acción del asistente.
6. El sistema valida que la fecha no sea futura y que todas las transferencias seleccionadas estén en estado "Hecho".
7. El sistema actualiza automáticamente: la fecha contable y la fecha de realización de las transferencias, la fecha de los movimientos de stock y sus líneas, la fecha de creación de las capas de valoración de inventario, y los asientos contables (los resetea a borrador, les asigna la nueva fecha y los vuelve a publicar).

**Reglas de negocio importantes:**

- **No se permiten fechas futuras:** Si la fecha ingresada es posterior a la fecha actual del sistema, el asistente mostrará un error y no procesará ningún cambio.
- **Todas las transferencias deben estar en estado "Hecho":** Si alguna de las transferencias seleccionadas no está completada, el sistema rechazará la operación completa con un mensaje de error.
- **Los asientos contables son restablecidos y reconfirmados:** El proceso lleva los asientos a estado borrador, les asigna la nueva fecha y los vuelve a publicar automáticamente, lo que puede generar una nueva secuencia de nombre si el período contable lo requiere.
- **La actualización de capas de valoración se realiza por SQL directo:** Dado que Odoo no permite modificar el campo `create_date` desde el ORM, el módulo utiliza una consulta SQL directa para actualizar este valor en las capas de valoración.
- **Los desechos (scraps) asociados a las transferencias también se incluyen:** Si existen registros de desecho vinculados a las transferencias seleccionadas, sus movimientos también son actualizados.

## Guía de Configuración

1. **Instalar el módulo** siguiendo los pasos descritos en la sección de instalación.
2. **Asignar usuarios al grupo de seguridad:** Ir a **Configuración → Usuarios y Compañías → Usuarios**, seleccionar el usuario deseado y en la sección de grupos (modo desarrollador activado) agregar el grupo **Allow to change date in picking**. Por defecto solo el Administrador y OdooBot tienen acceso.
3. **Verificar que el módulo `stock_account` esté activo:** Este módulo es un prerrequisito. Si no está instalado, la instalación de este módulo lo requerirá automáticamente.
4. **Advertencia sobre períodos contables bloqueados:** Si el período contable de la fecha destino está bloqueado, el proceso de reconfirmación del asiento contable puede fallar. Asegúrese de que el período contable esté abierto antes de ejecutar el cambio de fecha.
5. **Advertencia sobre secuencias de diario:** Al resetear y repostear los asientos contables, Odoo puede reasignar un nuevo número de secuencia. Confirme con el equipo de contabilidad si esto es aceptable antes de ejecutar la operación.

## Campos y Pantallas Clave

### Asistente para Cambio de Fecha en Transferencia

| Campo en pantalla | Para qué sirve | Valores posibles / Notas |
|---|---|---|
| Fecha | Define la nueva fecha y hora que se aplicará a la transferencia y todos sus registros relacionados (movimientos, asientos contables, capas de valoración). | Fecha y hora (datetime). No puede ser una fecha futura. Campo obligatorio. |

## Automatizaciones y Reglas

- **Validación de fecha futura:** Se ejecuta al confirmar el asistente. Si la fecha ingresada es mayor a la fecha actual del contexto del sistema, se lanza un error de usuario y se cancela toda la operación.
- **Validación de estado "Hecho":** Se ejecuta al confirmar el asistente. Si alguna de las transferencias seleccionadas no está en estado `done`, se lanza un error de usuario y se cancela toda la operación.
- **Actualización en cascada:** Al confirmar una fecha válida, el sistema propaga automáticamente el cambio a: transferencias (`accounting_date`, `date_done`), movimientos de stock (`date`), líneas de movimiento de stock (`date`), capas de valoración de inventario (`create_date` vía SQL), y asientos contables (reset a borrador → nueva fecha → reposteo).

## Preguntas Frecuentes (FAQ)

**P: ¿Puedo cambiar la fecha de una transferencia que todavía no ha sido validada (en estado Borrador o En Proceso)?**
R: No. El sistema solo permite cambiar la fecha de transferencias que estén en estado **Hecho**. Si selecciona transferencias en otro estado, el asistente mostrará un error y no realizará ningún cambio.

**P: ¿Puedo poner una fecha futura como nueva fecha de la transferencia?**
R: No. El módulo tiene una validación explícita que impide definir una fecha posterior a la fecha actual del sistema. Solo se pueden usar fechas pasadas o la fecha de hoy.

**P: ¿Qué pasa con los asientos contables cuando cambio la fecha de una transferencia?**
R: El sistema automáticamente lleva los asientos contables relacionados al estado borrador, les asigna la nueva fecha, resetea su nombre/secuencia y los vuelve a publicar. Es importante verificar que el período contable destino esté abierto antes de realizar esta operación.

**P: ¿Quién puede usar este asistente?**
R: Solo los usuarios que pertenezcan al grupo de seguridad **Allow to change date in picking**. Por defecto, únicamente el usuario Administrador y OdooBot tienen este permiso. Un administrador puede agregar más usuarios desde la configuración de usuarios en modo desarrollador.

**P: Si selecciono varias transferencias a la vez, ¿se les aplica la misma fecha a todas?**
R: Sí. La fecha ingresada en el asistente se aplica a todas las transferencias seleccionadas y a todos sus registros relacionados de forma simultánea.

**P: ¿El módulo también actualiza los registros de desecho (scraps) vinculados a la transferencia?**
R: Sí. Si existen registros de desecho asociados a las transferencias seleccionadas, sus movimientos de stock también son actualizados con la nueva fecha.

**P: ¿Puedo deshacer el cambio de fecha una vez realizado?**
R: El módulo no provee una función de deshacer. Para revertir el cambio, sería necesario ejecutar el asistente nuevamente con la fecha original. Se recomienda documentar la fecha anterior antes de realizar cualquier modificación.

**P: ¿El cambio de fecha afecta los reportes de valoración de inventario?**
R: Sí. Las capas de valoración de inventario (`stock.valuation.layer`) también son actualizadas con la nueva fecha mediante una consulta SQL directa, por lo que los reportes de costo de inventario por período reflejarán la fecha corregida.
